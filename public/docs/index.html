Introduction

This is a Delwathon purchase code verifier API documentation for eduthon product

Base URL: http://localhost

This documentation aims to provide all the information you need to work with our API.

As you scroll, you'll see code examples for working with the API in different programming languages in the dark area to the right (or as part of the content on mobile). You can switch the language used with the tabs at the top right (or from the nav menu at the top left on mobile).

Authenticating requests

To authenticate requests, include an Authorization header with the value "Bearer {YOUR_AUTH_KEY}".

All authenticated endpoints are marked with a requires authentication badge in the documentation below.

You can retrieve your token by authenticating via api/authenticate.

Endpoints

Authenticate.

This endpoint allows you to authenticate a user.

Example request:
curl --request POST \
    "http://localhost/api/authenticate" \
    --header "Content-Type: application/json" \
    --header "Accept: application/json" \
    --data "{
    \"secret_key\": \"eduthon_sk35-vGY1XVZcU88tgues\"
}"

const url = new URL(
    "http://localhost/api/authenticate"
);

const headers = {
    "Content-Type": "application/json",
    "Accept": "application/json",
};

let body = {
    "secret_key": "eduthon_sk35-vGY1XVZcU88tgues"
};

fetch(url, {
    method: "POST",
    headers,
    body: JSON.stringify(body),
}).then(response => response.json());
$client = new \GuzzleHttp\Client();
$response = $client->post(
    'http://localhost/api/authenticate',
    [
        'headers' => [
            'Content-Type' => 'application/json',
            'Accept' => 'application/json',
        ],
        'json' => [
            'secret_key' => 'eduthon_sk35-vGY1XVZcU88tgues',
        ],
    ]
);
$body = $response->getBody();
print_r(json_decode((string) $body));

Register new client.

This endpoint allows you to add a new client, then generate secret code for the client and a purchase code for the app.

Example request:

    "http://localhost/api/client" \
    --header "Content-Type: application/json" \
    --header "Accept: application/json" \
    --data "{
    \"firstname\": \"john\",
    \"lastname\": \"Doe\",
    \"email\": \"johndoe@example.com\"
}"

Verify client's purchase code.

requires authentication

This endpoint allows you to verify the purchase code inputed by the client and also checjk if the has been used on some other application.

Example request:
curl --request POST \
    "http://localhost/api/verify" \
    --header "Authorization: Bearer {YOUR_AUTH_KEY}" \
    --header "Content-Type: application/json" \
    --header "Accept: application/json" \
    --data "{
    \"purchase_code\": \"Oygfl-2yigH-VRWX8-dgEshd-RllUa\",
    \"client_id\": 3
}"

const url = new URL(
    "http://localhost/api/verify"

const headers = {
